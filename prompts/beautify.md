# 美化已有简历

用户已有一份简历，想做得更好看 + 更有料。流程：读 → 解析 → 诊断 → 经同意改内容 → 套模板。

## 步骤

### 1. 读取
- **PDF**：直接 Read（Claude 可读 PDF）。
- **Word/.docx**：让用户另存为 PDF 或复制粘贴文本（无法直接读二进制 docx 时）。
- **文本/Markdown**：直接读。

### 2. 解析进 schema
按 `schema/resume-data.md` 把现有简历拆成字段，缺失项标 `[缺失]`。

### 3. 诊断（对照 `guides/writing-tips.md`，给一份简短清单）
逐项检查并列出问题，**先诊断、先汇报，别闷头改**：
- **bullet 质量**：哪些是"职责罗列"（负责 X、参与 Y）而非"成果"（做了 X，带来 Z）？哪些缺量化数字？
- **动词**：开头是不是弱词（Responsible for / Worked on）？
- **结构**：板块顺序合不合适（资深却把 Education 放最前？应届却没突出项目？）。
- **长度/时间线**：是否超页、有无时间空档、格式是否统一。
- **ATS 风险**：原件是否用了多栏/文本框/图片承载关键信息（机器解析不到）。

把诊断写成一个清单给用户看，例如：
> "读完了，几个可以提升的点：① 你 3 条 bullet 没有量化结果；② '负责后端开发'是职责不是成果；③ 板块顺序建议把项目提前。我可以帮你逐条改，**只动表达不动事实**，要开始吗？"

### 4. 经同意再改内容
- **必须先问**再改。用户同意后逐条改写弱 bullet。
- **只改表达，不改事实**。需要补数字的地方问用户，拿不到标 `[待确认]`，绝不编。
- 改完逐条回读确认。

### 5. 套模板渲染
进 SKILL.md 的渲染流程。如果用户只想"换个好看的皮、内容先不动"，跳过第 4 步直接选模板渲染即可。
