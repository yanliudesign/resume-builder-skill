# 对话式建简历 — 访谈脚本

用户没有简历，通过一问一答帮他建。**一次只问一个问题**，顺着答案追问。目标是把内容填满 `schema/resume-data.md`，全程不杜撰。

## 开场

先说清节奏，降低压力：
> "好，咱们聊一聊就能出一份。我会一段一段问，你随便用大白话答，我来负责整理成专业表达。先从最简单的开始——你想找的是什么岗位？"

## 采集顺序（一段一段走，挖完一段再下一段）

### 1. 目标 & 联系方式
- 目标岗位 / headline（决定全篇取舍和关键词）
- 姓名、邮箱、城市；可选电话
- LinkedIn / GitHub / 作品集链接（有就要）

### 2. 经历（核心，逐段挖）
对每一段工作，按这个三步追问，**一段做完再问下一段**：
1. **基本信息**："这段是哪家公司、什么职位、从什么时候到什么时候？"
2. **做了什么**："这个岗位你主要负责什么？挑 2-3 件你最有成就感 / 最花精力的事说说。"——用户用大白话讲，你来提炼。
3. **成果追问**（关键）："这件事最后带来了什么变化？有没有数字——比如提升了多少、省了多少时间、覆盖多少用户？"
   - 用户说不出数字时，引导而非编造："那当时是从无到有吗？影响了多大范围的人？大概什么量级？"——拿不到就标 `[待确认]`，**不替他填**。

对每件事，当场转成一条 bullet（动词开头 + 做了什么 + 量化结果），回读给用户："我帮你写成这样——『…』，对吗？数字准不准？"

### 3. 项目（如果经历薄 / 学生 / 转行）
同样的三步：是什么项目、你做了什么、结果/规模/链接。

### 4. 教育
学校、学位+专业、毕业时间；问一句要不要放 GPA / 荣誉 / 相关课程（应届才建议放）。

### 5. 技能
"还有哪些硬技能 / 工具 / 语言要放上去？"——帮他按 Languages / Frameworks / Tools 之类分组。

### 6. 可选板块
快速扫一遍要不要：证书、获奖、语言、出版。有才问细节，没有跳过。

## 收尾

1. 把整理好的 `resume-data` 完整回显，**把所有数字单独列出来让他逐个确认**。
2. 确认后进入 SKILL.md 的渲染流程（选模板 → 出 HTML）。

## 反模式（别犯）

- ❌ 一口气问"告诉我你的姓名、岗位、所有经历和技能"——太重，回到一次一问。
- ❌ 用户没给数字，自己脑补一个"提升 30%"——只能 `[待确认]`。
- ❌ 把用户的大白话原样塞进 bullet——要提炼成"动词+成果"，但提炼完必须回读确认。
