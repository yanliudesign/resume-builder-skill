# 简历写作 Cheat Sheet

渲染前用这套标准检查/改写内容。核心：**每条 bullet = 强动词 + 做了什么 + 量化结果。**

## Bullet 公式

```
[动词] + [具体做了什么] + [带来的可量化结果]
```

- ✅ "Rebuilt the checkout API, cutting p95 latency from 1.2s to 380ms and lifting conversion 8%."
- ❌ "Responsible for the checkout API."（职责，无成果，无数字）
- ✅ "重构结算 API，将 p95 延迟从 1.2s 降到 380ms，转化率提升 8%。"

每条尽量答出：**做了什么 → 怎么做的 → 结果如何（最好有数）。** 没有数字也要有"影响/范围"。

## 强动词（开头别用 Responsible for / Worked on）

Built, Led, Shipped, Designed, Launched, Cut, Increased, Reduced, Automated, Scaled, Migrated, Owned, Drove, Negotiated, Mentored, Resolved.
中文：搭建、主导、上线、设计、推动、降低、提升、自动化、迁移、负责（少用）、攻克、带教。

## 量化的来源（用户答不出数字时这样引导）

- 百分比：提升/降低了多少 %？
- 绝对量：多少用户 / 请求 / 收入 / 行代码 / 人的团队？
- 时间：省了多少小时 / 周期从多久缩到多久？
- 范围：覆盖几个国家 / 几条产品线 / 多大规模？

引导但**绝不替用户编**。拿不到 → `[待确认]`，或写定性影响（"成为团队主用工具"）。

## 结构与长度

- **倒序**：最近的经历/教育在最上。
- **一页优先**：<10 年经验控制 1 页；资深可 2 页。
- **板块取舍**：应届/转行把 Projects + Education 提前；资深 Experience 第一、Education 末尾一行。
- **时态**：现职用现在时，过往用过去式（英文）。
- **去人称**：英文 bullet 不写 "I / My"。

## ATS 友好（海投大公司尤其重要）

- 单栏、标准板块标题（Experience / Education / Skills），机器才解析得对。
- 别把关键信息塞进图片、文本框、页眉页脚。
- **抄岗位 JD 里的关键词**（技能、工具、职位名）——匹配度直接影响过筛。
- 联系方式用纯文本，别只放图标。
- 用 `templates/classic-ats.html` 时这些已经满足；花哨模板适合定向投递/人会直接看的场景。

## 常见错误清单（诊断时逐项对照）

- [ ] bullet 是职责堆砌而非成果
- [ ] 缺量化数字
- [ ] 弱动词开头
- [ ] 板块顺序不匹配资历
- [ ] 超页 / 时间线有空档没解释
- [ ] 格式不统一（时间格式、标点、字体）
- [ ] 关键信息藏在图片/多栏里（ATS 解析不到）
- [ ] 有错别字 / 时态混用
- [ ] 放了无关老旧经历挤掉了重点
